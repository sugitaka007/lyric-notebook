# アートメモ

「アートメモ」は、歌詞・言葉・メロディ・音・映像の思いつきをすばやく記録する、iPhone向けの個人用メモアプリです。入力したメモは、あとから曲へ整理できます。

曲の画面は「歌詞・アイデア・写真・録音・音声・スケッチ」に分かれています。「録音」はSafariでその場で録る音、「音声」はボイスメモなど既存ファイルの取り込みです。クイック追加とアイデアのフレーズは、定型の依頼文とまとめてクリップボードへコピーできます。API送信は行いません。

スケッチでは線・文字・矢印・三分割ガイドを使ってMVの構図を記録できます。画像生成用の条件を文章へまとめてコピーし、スケッチ自体はPNG画像として保存できます。

- 月額料金・従量料金・独自ドメイン料金は不要です。
- OpenAI APIを含む有料API、バックエンド、クラウドDB、広告、アクセス解析は使いません。
- 個人データは外部へ送信せず、SafariのIndexedDBに保存します。
- 一度読み込めばオフラインでも起動できるPWAです。
- サンプル曲や例文は作成しません。初回起動時も空の状態から始まります。

## 1. パソコンで起動する

Node.js 20以上をインストールしてから、このフォルダでターミナルを開きます。

```bash
npm install
npm run dev
```

ターミナルに表示された `http://localhost:...` をブラウザで開きます。終了はターミナルで `Ctrl + C` です。

このリポジトリには `pnpm-lock.yaml` も含まれています。pnpmを使う場合は `pnpm install`、`pnpm dev` でも起動できます。

## 2. 開発中の確認

```bash
npm run lint
npm test
npm run build
npm run preview
```

- `lint`: 書き方の問題を検査します。
- `test`: IndexedDB、関連付け、並べ替え、バックアップ、DB移行を検査します。
- `build`: GitHub Pagesへ置ける静的ファイルを `dist` に作ります。
- `preview`: 完成版をパソコン内で確認します。

## 3. GitHub Pagesへ無料公開する

必要なのは無料のGitHubアカウントだけです。クレジットカード、有料プラン、独自ドメインは不要です。

1. GitHubで新しいリポジトリを作ります。名前は `lyric-notebook` などで構いません。
2. 無料利用を確実にするため、リポジトリは `Public` を選びます。公開されるのはアプリのソースだけです。iPhone内の歌詞や画像はリポジトリへ送られません。
3. このフォルダのファイルをリポジトリの `main` ブランチへアップロードします。`node_modules` と `dist` はアップロード不要です。
4. GitHubのリポジトリで `Settings` → `Pages` を開きます。
5. `Build and deployment` の `Source` で `GitHub Actions` を選びます。
6. `Actions` タブの「GitHub Pagesへ公開」が緑のチェックになるまで待ちます。
7. `Settings` → `Pages` に表示されたURLをiPhoneのSafariで開きます。通常は `https://ユーザー名.github.io/リポジトリ名/` です。

リポジトリ名に応じたパスは自動設定されます。`ユーザー名.github.io` という特別なリポジトリ名の場合も自動で `/` に切り替わります。

## 4. iPhoneのホーム画面へ追加する

1. 公開URLをSafariで一度開き、画面が表示されるまで待ちます。
2. Safari下部の共有ボタン（四角から上向き矢印）を押します。
3. 「ホーム画面に追加」を押します。
4. 名前を確認して右上の「追加」を押します。

2回目以降は追加した「アートメモ」アイコンから開けます。オフライン確認は、一度オンラインで開いた後に機内モードへ切り替え、ホーム画面のアイコンから起動してください。

## 5. バックアップと復元

### バックアップ

1. ホーム画面右上の「設定」を開きます。
2. 「データ管理」を開き、「バックアップを書き出す」を押します。
3. iPhoneの共有メニューで「ファイルに保存」を選びます。
4. 保存先に `iCloud Drive` を選びます。

ZIPの中には `manifest.json`、`data.json`、画像・音声・スケッチの実ファイルが入ります。形式バージョンと作成日時も記録されます。

### 復元

1. ホーム画面右上の「設定」を押します。
2. 「データ管理」を開き、「バックアップから復元」でファイルアプリ内のバックアップを選びます。
3. 検証結果（日時、曲数、素材数）を確認します。
4. 「追加」または「置き換え」を選びます。
5. 置換の場合は、もう一度表示される確認を読んでから実行します。

「既存へ追加」はIDを新しく割り当てるため、同じバックアップを読み込んでも既存曲を上書きしません。

## 6. データが保存される場所

歌詞、画像、録音、スケッチなどの主要データは、iPhoneのSafariが公開URLごとに管理するIndexedDBへ保存されます。`localStorage` を主要データ保存には使っていません。テーマなどの設定もIndexedDBへ保存します。

ページ再読込や通常のアプリ終了ではIndexedDBを削除しません。今回の簡略版へ初めて更新するときだけ、利用者の指定により旧バージョンの曲・歌詞・素材・未整理メモを一度だけ消去します。その後の通常起動でデータを消すことはありません。必要な旧データは、以前に作成したZIPバックアップを手動で復元できます。

## 7. Safariデータ削除時の重要な注意

次の操作ではデータが消える可能性があります。

- iPhoneの「設定」→「Safari」から履歴とWebサイトデータを削除する
- 「詳細」→「Webサイトデータ」でGitHub Pagesのデータを削除する
- iPhoneの空き容量が極端に少なくなり、Safariが保存領域を整理する
- 別のGitHub Pages URLへ移動する

SafariのIndexedDBはiCloud同期されません。定期的にZIPバックアップを作り、iCloud Driveへ保存してください。アプリは2週間以上バックアップしていない場合に控えめな注意を表示します。

## 8. アプリを更新する

修正したファイルをGitHubの `main` ブランチへ追加すると、GitHub Actionsが自動で再公開します。PWAは次回起動時に新しい静的ファイルへ更新されます。IndexedDBは静的ファイルとは別の領域にあるため、更新だけで曲データは消えません。

大きな変更の前には、念のためZIPバックアップを作ってください。

## 9. プライバシーと料金

実行中の通信は、GitHub Pagesからアプリ本体のHTML・CSS・JavaScript・アイコンを取得するためだけです。歌詞、画像、録音、スケッチ、バックアップ内容を送信する `fetch`、APIクライアント、フォーム送信処理はありません。

使用していないもの：OpenAI API、オンライン同期、共同編集、ユーザー登録、SNS共有、オンラインフォント、CDN、外部画像サービス、広告、アクセス解析、独自ドメイン。

GitHub Pagesの無料枠で公開する限り、月額料金・従量料金は発生しません。

## 10. iPhone / Safari上の制限

- 録音形式はSafariの対応状況により `audio/mp4` または `audio/webm` になります。録音できない場合はボイスメモで録音し、「音声ファイル」から追加できます。
- iPhoneの空き容量やSafariの判断により、利用できる保存上限は一定ではありません。
- PWAの初回オフライン起動には、オンラインで一度最後まで読み込む必要があります。
- iCloud Driveへの保存は、Safariの共有メニューで本人が保存先を選ぶ必要があります。アプリがiCloudへ自動送信することはありません。
